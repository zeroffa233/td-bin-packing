# 入参

> 每个订单对应一次请求，一次请求涉及如下的一次入参和出参。所以我们下面的内容都是基于一次请求进行说明，并且在一次请求中，可能涉及到把物品装到多个箱子中，需要多次调用算法。
> 并且有些参数是不需要参与算法逻辑和计算的，我在该参数的说明中会标明 not_important，该部分参数如果出参中要用到，就原样传出，如果没有用到，就不用管。
> 并且部分参数我合在同一个标题中说明，不代表三者位于同一个结构体中，请你注意对照入出参文档。

## items

类型：array

items 的意思是这次请求涉及的所有需要装箱的物品。

### item

类型：object

#### code

类型：string

该物品的编码，与下面的 order 参数里的 itemCode 关联。

#### itemCategory 1

类型：string

可能值：LH，正品，赠品，小样

涉及后面的具体调度逻辑，我们管这个叫商品的主类别，和我们二段装箱中的礼盒逻辑是一样的：我们需要优先把正品装在礼盒里面，然后把LH，小样和赠品作为散件再装入 container，我们保证LH可以装下所有的正品。

#### itemCategory 2

类型：string

可能值：易碎

涉及后面的具体调度逻辑，我们管这个叫商品的附加类别，但是这个目前还不知道有什么用，先不用管。

#### weight

类型：decimal

涉及可能存在的箱子重量限制校验。

#### length，width，height

类型：decimal

物品的尺寸，我们的离散算法可能需要有一个入参可以指定精度缩放。

#### csQty

类型：bigint

这个参数非常重要，order 中的物品数量必须先对这个数（整箱的数量，不需要调度）取余，余数（散件）才是我们真正需要传入核心算法进行调度的数量。

如果是 None，则不做取余处理。

#### allowDown

类型：int

0=不允许、1=允许、只有允许放倒才可任意摆放，否则必须向上摆放，换言之物品只能在 z 轴上进行旋转，可能的姿态变为 2 种。

#### volume

not_important

原则上我们可以用这个体积进行缓存，但是避免和尺寸信息出现冲突，我们算法逻辑中只使用长宽高实时计算体积，不使用这个值。

#### name，brand，itemSize，itemColor

not_important

## containers

类型：array

### container

类型：object

#### code

类型：string

箱子的编码，用来出参时标识箱子型号。

#### length，width，height

类型：decimal

箱子的尺寸信息，用来供算法计算。

#### maxWeight

类型：decimal

当该值传入的不是 None 的时候，就需要启用装箱后的后合法性校验——重量，在装箱算法运行完成后，校验是否超重，超了就需要换个箱子。

#### capacityRate

类型：decimal

可用容量百分比，该值涉及到装箱后的后合法性校验——体积，在装箱时不用管这个值，我们认为箱子空间是可以完全利用的，但是装好后需要把利用率和这个百分比做对比，超过了我们认为就不合法，因为没有留出泡泡纸等物品的空间。

#### sequence

类型：int

序号越小，优先级越高，我们在尝试算法的时候，按照(序号, 体积)的字典序升序尝试。

#### quantity

类型：bigint

我们在每次请求中可能是多次运行算法，因为可能需要分多个箱子装，每次请求需要保证箱子是有剩余的。

#### stores

类型：string

用来做箱子的前合法性校验，该箱子可以使用于某个物品当且仅当 orderDetail （一个orderDetail 对应一种商品）中的 stores 等于该值，或者两个值中有一个是 None。

#### name

not_important

## order

类型：array

一个 order 是一个订单，对应一次入参，即一次请求。

### orderDetail

类型：object

一个 order 由多个 orderDetail 组成，因为一个 order 中包含很多商品。

#### shipmentDetailId，shipmentCode，waveId，itemName，isTaoX

not_important

#### itemCode

类型：string

对应一种商品，通过和 item 中的 code 属性关联，可以获得商品信息。

#### quantity

类型：bigint

订单数量，需要和对应 item 的 csQty 做取余操作，获得实际进入算法的数量。

#### batch，lot，manufactureDate，buildCode，zoneCode

类型：string

做 lclRules 的拼箱规则校验时用，到那部分再说明。

#### stores

类型：string

说明该物品只能用 container 的 stores 字段等于该字段的 container 装，在一个订单的同一个 groupId 内，可以所有 items 认为 stores 是一样的。

#### carrierService

类型：int

可能值：0=快递、1=物流

当该值等于 0 的时候，针对该次请求执行我们的正常的算法计算最优装法。

当该值等于 1 的时候，直接选择对应 stores 的箱子中体积最小的那个返回。

注意该值是针对该次请求即最顶层的入参而言的，输入数据可以保证所有的 orderDetail 中该值都相等，所以只需要针对第一条 orderDetail 校验一次。

#### groupId

同一个 groupId 内进行

## lclRules

类型：array

该部分是最复杂的部分，该参数决定了我们所有的散件要按照什么样的决策顺序装在多个箱子中。我们会在该部分给出具有优先顺序的多个规则(<=6个)，在装箱时，我们需要按照优先级按照下面的算法逻辑装：

1. 选定当前没考虑过的优先级最高的规则作为当前规则
2. 以当前规则筛选出符合要求的商品子集，并给出标志位（1代表商品子集是剩下的所有商品了，0代表还有商品）
3. 尝试把该商品子集调度到某个箱子中，如果标志位是 0，就需要满足：该箱子刚好装不下这么多商品（意思是换再低一级优先级或者再大一号的箱子就装的下了）；如果标志位是 1，就需要满足：该箱子刚好可以装下所有商品，即传统的装箱算法（可以运行多次以装下这些商品）。如果标志位是 0 且不存在这样的箱子，就把规则切换到低一优先级的规则，回到第二步。如果标志位是 0 且不存在这样的箱子，也不存在低一优先级，就把剩下的所有的商品（包括非该子集内的商品）直接装好（选择刚好能装下的箱子，可以运行多次算法装在多个最大的箱子中），出参的 code 返回 1 （失败）。
4. 把该箱子的装法定下来，装好的商品在商品集中删除
5. 回到第二步

### lclRule

类型：object

表示启用了哪几个拼箱规则，一个 lclRule 代表了一种拼箱规则。

#### sequence

类型：int

该规则在此次请求中的优先级。

#### code

类型：string

分为六个类型：

1. SAME_SKU_LOT 同品同批次（批次由批次批号生产日期合并组成）同箱

    商品子集为 itemCode + batch + lot + manufactureDate 均相同的最大集合。

2. SAME_SKU 同品同箱

商品子集为 itemCode 相同的最大集合。

3. SAME_TYPE 同款同箱

商品子集为 itemStyle

4. SAME_BUILD_ZONE 同建筑体同区域同箱

商品子集为 buildCode + zoneCode 相同的最大集合。

5. SAME_BUILD 同建筑体同箱

商品子集为 buildCode 相同的最大集合。

6. SAME_ZONE 同区域同箱

商品子集为 zoneCode 相同的最大集合。

6. NO_LIMIT 不限制

    商品子集为所有商品。

#### name

not_important

## unpackJudge

类型：int

可能值：1=允许用多个箱子装，0=不允许用多个箱子装

虽然该参数中指定了是不是允许用多个箱子装，但是我们无论如何都要正常执行使用多个箱子的算法，然后计算是不是使用了多个箱子的多箱标志位 ifMultiContainers，假设使用了多个箱子是 1，只使用了一个箱子是 0，最后出参中错误码返回逻辑表达式： (unpackJudge 同或 ifMultiContainers) 或 unpackJudge。

## code

类型：int

可能值：0=成功，1=失败

状态码，需要这样计算：如果拼箱规则环节失败，则直接按照规则返回 1 ，如果拼箱环节成功，需要按照 ~unpackJudge && ifMultiContainers 规则返回。

其余环节可能有非法情况，也一票否决式地返回 1。

## msg

类型：string

异常信息等内容。

## data

类型：object

返回的调度结果。

### infos

类型：array

由于拼箱结果很复杂，所以我们使用类似 order 的多行结构来描述。

#### info

类型：object

每行 info 表示特定箱子中的某种商品。

##### shipmentDetailId，shipmentCode，waveId

类型：bigint

按照 not_important 输入原样输出。

##### itemCode

类型：string

商品编码。

##### itemName

类型：string

商品名。

##### quantity

类型：bigint

该容器内的数量。

##### batch，lot，manufactureDate，buildCode，zoneCode，stores

类型：string

按照该物品订单的输入原样输出。

##### containerSequence

类型：int

箱子顺序号，装箱时从 1 开始计数。

##### containerCode

类型：string

该箱子的型号编码。
